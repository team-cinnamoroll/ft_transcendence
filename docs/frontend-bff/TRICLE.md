## サービスコンセプト

「自分が主体」のアクティビティ記録サービス。Twitter のような「つながり・発信・会話」ではなく、**自分の好きなことを気兼ねなく書き留める**ことを主目的とする。いいね・メンション・リプライは意図的に提供していない。

### Twitter との対応表

| Twitter | Trickle |
|---|---|
| ツイート | アクティビティ |
| フォロー | サブスクライブ |
| 引用RT | リンク |
| 複数アカウント | トピック |

## 主要概念

### トピック

- ユーザーが自由に作成する**カテゴリ**（ブログのカテゴリに相当）
- 必ずトピックごとにアクティビティを書く（トピックなしでは投稿できない）
- 例: 「読書」「映画」「ゲーム」「日記」など
- トピックはユーザー単位で独立（サービス全体で共通ではない）

### アクティビティ

- トピックに書き込むメモ・投稿（Twitterのツイートに相当）
- **文字数上限: 5000文字**
- 長文は折りたたんで表示
- 画像添付可（動画は未対応）
- 編集・削除自由、時間制限なし

### サブスクライブ

- 「人をフォロー」ではなく「**特定ユーザーの特定トピックをサブスクライブ**」する
- 例: 「Aさんの読書トピックはサブスクするが、ゲームトピックはしない」
- サブスクしたトピックに更新があると**プッシュ通知**が来る（通知OFF機能なし）
- サブスク中のアクティビティはホームとは**別タブ**に表示

### リンク

- 自分のアクティビティに他のアクティビティを関連付ける機能（Twitterの引用RTに相当）
- 複数のアクティビティにリンク可能
- リンクしたアクティビティ一覧・リンクされたアクティビティ一覧を閲覧可能（他ユーザーも閲覧可能）

## 画面構成（タブ）

| タブ | 内容 |
|---|---|
| ホーム | 自分が書き留めたアクティビティの一覧 |
| サブスク | サブスクライブ中のトピックのアクティビティ一覧 |
| 検索 | アクティビティ・トピックの検索 |
| プロフィール | 自分のトピック一覧・振り返り機能など |

## 機能一覧

### 振り返り機能

- **タイル表示**: GitHubのコントリビューション風に日々の投稿数を可視化。多い日ほど濃い緑
- タイルをタップするとその週のアクティビティを確認可能
- **年間データ機能**: 年間のアクティビティ数・投稿日数を表示。トピック別絞り込みも可能

### カラーテーマ

- 14色 × ライト/ダーク = **28パターン**
- カラフルテーマ（ダーク）では背景がユーザーアイコンやトピック画像になる

### バッジ

- ユーザー名の後ろに表示できる装飾
- **600種類以上**から選択可能

### 検索

- アクティビティとトピックを検索
- 検索スコープ: 「サービス全体」「自分のアクティビティ」「サブスクしているトピック」から選択可能
- 内部的には文字列の部分一致（Like検索）

### ユーザーブロック

- 特定ユーザーをブロック可能
- ブロック後はお互いに以下が不可になる:
  - アクティビティの閲覧
  - アクティビティへのリンク
  - トピックのサブスクライブ

### Web公開

- 自分のアクティビティ・トピックを Web URL で公開可能
- アクティビティ単位 or トピック丸ごとで公開設定が可能
- ウェブ版はログイン・書き込み機能なし（閲覧専用）

### ログイン

- **Googleログイン**のみ対応（iOS は Apple ログインも可）

## 対応プラットフォーム

| プラットフォーム | 用途 |
|---|---|
| スマホ（iOS/Android） | 書く・読む含めすべての機能。メインプラットフォーム |
| デスクトップ（Windows/Mac） | 書くことに特化。他ユーザーのアクティビティ閲覧は不可 |
| Web | 公開されたアクティビティ・トピックの閲覧専用 |

## ない機能（意図的に非提供）

| 機能 | 理由 |
|---|---|
| いいね / メンション / リプライ | コミュニケーションを強く誘発する機能は提供しない方針 |
| 鍵アカウント（承認制サブスクライブ） | サービスのコンセプト（コミュニケーション重視でない）に合わないため |
| プッシュ通知のOFF設定（トピック単位） | 「本当に見たいものだけサブスクする」というサービス設計意図のため |
| DM | 電気通信事業の届け出が必要になるため |

## ユーザーの声まとめ

- いわゆる壁打ち（Twitterで自分の考えや気持ちをひとりごとのように吐き出し続けるやつ）が好きな人には向いてます。

- 私のアクティビティで1番多いのはDuolingoのメモです。まちがえた例文、覚えたい例文、連続記録のスクショなど。

- 表通りの賑やかさとは一線を画す、静かで奥深いサービス。

- 思考の断片やひらめきを少しずつ、まるで水滴のように貯めていく――そんなユニークな体験を提供してくれるアプリ。

- 私自身、普段から様々な情報に触れ、頭の中でたくさんのアイデアが生まれては消えていくことに悩んでいました。ブログのネタ、企画の構想、ふとした瞬間の気づき…。「あ、今いいこと思いついた！」と思っても、すぐに別のことに意識が向いてしまい、結局記憶の彼方へ。そんな私にとって、Trickleはまさに救世主のような存在でした。

- 「裏路地スタイル」の次の記事のテーマが浮かんだら、すぐに「ブログネタ」というトピックを作成（または既存のトピックに投稿）し、箇条書きでキーワードや構成案を書き出します。長文を書く必要がないので、移動中やちょっとした待ち時間にもサッと書き込めるのが非常に便利です。

- 最近読んだ本や記事、気になったWebサイトなどから得た情報を、特定のトピックにまとめています。例えば、「読書メモ（ビジネス）」というトピックを作成し、読んだ本の重要なフレーズや、そこから得た気づきを書き残します。

- Webサイトの場合はURLを貼り付け、そのサイトの何が面白かったのか、どんな点が参考になったのかを簡潔にメモします。これにより、後から特定のテーマについて調べたい時に、自分が過去にインプットした情報がまとめて確認できます。まるで自分専用の「情報キュレーションサービス」のような感覚で活用しています。

- 仕事やプライベートで複数のプロジェクトを抱えている場合、それぞれのプロジェクトごとにトピックを作成し、進捗状況や次にやるべきこと、あるいは思いついた新しいアイデアを「水滴」として残しています。

- これは少し抽象的な使い方ですが、Trickleは私にとって「思考の整理」と「セルフディスカバリー（自己発見）」の場でもあります。
特にテーマを決めずに、その時頭に浮かんだことをひたすら書き出す「ジャーナリング」のような使い方をすることもあります。一見、まとまりのない思考の羅列に見えるかもしれませんが、後から見返すと、自分が今何に興味を持っているのか、どんなことに悩んでいるのか、どんなことに喜びを感じているのかなど、自分自身の内面を客観的に見つめ直すことができます。
バラバラだった「水滴」が、やがて自分自身の思考の「流れ」を形成していく。このプロセスを通じて、これまで気づかなかった自分の価値観や、新たな興味の方向性が見えてくることがあります。これは、デジタルツールでありながら、どこかアナログな「内省」に近い体験だと感じています。

- 3Dモデリングについてやイラストなど、趣味の創作についてのトピックを使っています。自分の感想なども残しておくと、この時はこういう風に考えていたのかというのが、不意に後から追いかけられたりしてなかなか面白いです。

- かわいい画像が流れてくるトピックなどをサブスクライブしています。
