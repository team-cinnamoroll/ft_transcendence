# 仕様変更

## サービス名

**「MultiFace」**

### 表記ルール

| 用途 | 表記 |
|---|---|
| 表示名・ロゴ・UI テキスト | `MultiFace` |
| コード上の識別子（変数・クラス名など） | `multiFace` / `multiface` |
| URL スラッグ・ファイル名 | `multiface` |

> 「Multi Face」（スペースあり）・「multi-face」（ハイフンあり）は使用しない。

## サービスコンセプト

- 旧Twitter: 当初のコンセプトは**日常のつぶやき**。短文・リアルタイム性を特徴とする、情報収集と拡散に優れたSNS。

- 従来のTricle: 「自分が主体」のアクティビティ記録サービス。Twitter のような「つながり・発信・会話」ではなく、**自分の好きなことを気兼ねなく書き留める**ことを主目的とする。いいね・メンション・リプライは意図的に提供していない。

- MultiFace: **フェイス数=ユーザーの多面性**。ユーザーは多面性の数だけフェイスを持ち、その時々で適したフェイスにアクティビティを記録する。Tricleのコンセプトである**自分の好きなことを気兼ねなく書き留める**を踏襲しつつ、新しいコンセプトで独自の価値を提供する。

### Twitter、Tricleとの対応表

| Twitter | Trickle | MultiFace |
|---|---|---|
| ツイート | アクティビティ | アクティビティ |
| フォロー | サブスクライブ | サブスクライブ |
| 引用RT | リンク | リンク |
| 複数アカウント | トピック | フェイス |

## 主要概念

### フェイス

- ユーザーが自由に作成する**多面性の一つ**
- ユーザーは投稿したい内容にあったフェイスで投稿する
- 例: 「読書」「映画」「ゲーム」「日記」など
- フェイスはユーザー単位で独立（サービス全体で共通ではない）
- 作成できるフェイス数に上限はない
- **公開/非公開**を設定可能。非公開フェイスは自分のみ閲覧でき、他ユーザーからはフェイス自体が存在しないように見える（検索・サブスクライブ不可）

### アクティビティ

- フェイスに書き込む投稿（Twitterのツイートに相当）
- **文字数上限: 5000文字**
- 長文は折りたたんで表示
- 画像添付可（最大4枚・動画は未対応）
- 編集・削除自由、時間制限なし

### サブスクライブ

- 「人をフォロー」ではなく「**特定ユーザーのフェイス（多面性の一つ）をサブスクライブ**」する
- 例: 「Aさんの読書フェイスはサブスクするが、ゲームフェイスはしない」
- サブスクしたフェイスに更新があると**プッシュ通知**が来る（通知OFF機能なし）
- サブスク中のアクティビティはホームとは**別タブ**に表示

### リンク

- 自分のアクティビティに他のアクティビティを関連付ける機能（Twitterの引用RTに相当）
- **自分・他ユーザー問わず**任意のアクティビティにリンク可能
- 複数のアクティビティにリンク可能
- リンクしたアクティビティ一覧・リンクされたアクティビティ一覧を閲覧可能（他ユーザーも閲覧可能）
- リンクされた投稿者へは**アプリ内通知のみ**（プッシュ通知は来ない）

#### リンク元アクティビティが削除された場合の挙動

| 場所 | 挙動 |
|---|---|
| 通知一覧 | 通知自体は「過去の出来事のログ」として残る |
| 通知→リンク先への遷移 | 「このアクティビティは削除されました」エラーを表示。内容は閲覧不可 |
| 自分のアクティビティ詳細の「リンク一覧」 | 削除されたリンク元は自動的にリストから消去される |

## 画面構成（タブ）

| タブ | 内容 |
|---|---|
| フェイス一覧 | 自分のフェイスの管理（作成・編集・削除） |
| ホーム | 画面上部にフェイスのアイコン一覧。下部からは自身の全フェイスのアクティビティ一覧 |
| サブスクライブ | サブスクライブ中の他ユーザーのフェイスのアクティビティ一覧 |
| 通知 | サブスクライブ中のフェイスで新規のアクティビティが投稿されたら通知 |
| 検索 | アクティビティやフェイスを検索する |

### フェイス一覧

- 自身のフェイスを**管理（作成・編集・削除）**するタブ。
- アクティビティが最新のフェイスから表示する。
- 右下にフェイスの新規追加FABを配置
- FABではフェイス写真、フェイス名、フェイス詳細、公開/非公開を設定する。

### ホーム

- 上部には自身のアカウントのアイコン、フェイス数、アクティビティ数、タイル表示（GitHubのコントリビューション風に日々の投稿数を可視化。多い日ほど濃い緑）等のプロフィール的な情報を表示する（独立したプロフィールページは持たない）。
- 上部のフェイスアイコン一覧をタップすると**そのフェイスでアクティビティを絞り込む**（フィルタ操作）。
- 中部からはTwitterのタイムライン的な感じで自身のアクティビティ一覧を表示（スクロール可）
- 右下にアクティビティの新規追加FABを配置
- FABではフェイス選択、アクティビティ本文、画像添付ボタンを設定する。

### サブスクライブ
- 検索などでサブスクライブしたフェイスのアクティビティ一覧をタイムライン的に確認できる。
- 検索などでサブスクライブしたフェイス一覧を確認できる。
- **サブスクライブしているフェイスが0件の場合**、おすすめフェイスや人気フェイスを提示するオンボーディング UI を表示し、検索への導線を設ける。

### 通知

アプリ内の「お知らせ（通知一覧）」に表示されるのは、主に以下の2点です。

- リンク（Link）: 誰かがあなたのアクティビティを引用（リンク）した際。
- サブスクライブ（Subscribe）: 誰かがあなたのフェイスを購読開始した際。

また、他の人との関わりはサブ的な要素であるため、サブスクライブされたりリンクされてもアプリ内のお知らせのみでプッシュ通知はきません。

### 検索
- アクティビティやフェイスを検索することができる。
- 検索については「サービス全体から」「自分のアクティビティから」「サブスクしているフェイスから」の３種類で検索可能。
- 「全体から」で検索すると自分が興味あるフェイスを新たに発見でき、いいと思ったらサブスクライブする。

## 機能一覧

### プッシュ通知
- サブスクライブしているフェイスに投稿があるとプッシュ通知が届きます。これは「プッシュ通知でみたいぐらい本当に関心があるフェイスだけをサブスクライブする」という設計だからです。

### ユーザーブロック
- 特定ユーザーをブロック可能
- ブロック後はお互いに以下が不可になる:
  - アクティビティへのリンク
  - フェイスのサブスクライブ（新規）
- **閲覧制限**: ブロックすると相手はあなたのフェイス・アクティビティを閲覧できなくなる
- **既存サブスクライブの扱い**: ブロック前にサブスクライブ済みであっても自動解除はしない。相手の「サブスクライブ一覧」にはフェイスが残るが、アクティビティは表示されない
- **通知の遮断**: ブロック後はあなたの新しい投稿が相手のサブスクライブフィードに流れない

## ない機能（意図的に非提供）

| 機能 | 理由 |
|---|---|
| いいね / メンション / リプライ | コミュニケーションを強く誘発する機能は提供しない方針 |
| 鍵アカウント（承認制サブスクライブ） | サービスのコンセプト（コミュニケーション重視でない）に合わないため |
| プッシュ通知のOFF設定（フェイス単位） | 「本当に見たいものだけサブスクする」というサービス設計意図のため |
| DM | 電気通信事業の届け出が必要になるため |
| プロフィールページ（独立した画面） | フェイス重視のコンセプトに合わないため。アカウント情報はホーム上部にまとめて表示する |
